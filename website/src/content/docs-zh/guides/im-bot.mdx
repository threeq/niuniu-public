牛牛的 IM 机器人让你**不打开桌面 app，直接在飞书 / 钉钉 / Telegram 群里驱动 agent 工作**：在群里说一句需求，牛牛在对应项目里起 workspace 让 agent 执行，过程与结果再对话式回到群里。

核心是一句话：**一个机器人（一份凭据、一条连接）即可把不同群分流到你名下的不同项目**。

## 支持的平台

| 平台 | 连接方式 | 入站 | 出站 |
|---|---|---|---|
| **飞书 / Lark** | 长连接（无需公网回调） | 文本、图片、附件 | markdown、回执、任务状态 emoji |
| **钉钉** | 长连接 | 文本、图片、附件 | markdown、回执、任务状态 emoji |
| **Telegram** | Bot API | 文本、图片、附件 | markdown、回执、新建回复 |

## 第一步：新建机器人

入口：**设置 → IM 机器人 → + 新建机器人**。

1. 选平台（飞书 / 钉钉 / Telegram），填入该平台开放平台申请到的机器人凭据（App ID / Secret 或 Bot Token 等，按平台字段提示）。
2. 保存后机器人以**长连接**方式在线，状态显示 **已启用**；随时可点 **测试** 验证连通。

一个机器人可以服务多个群、多个项目——**不必为每个项目各建一个机器人**。

## 第二步：把群绑定到项目

机器人进群后，需要告诉牛牛"这个群里的消息应该落到哪个项目"。

- 在机器人的**待配对群**列表里，选中一个群，指定**目标项目**，完成绑定。
- 绑定成功后，该页会展示这条群的 `channel_id`，方便你回填或排查。
- **多项目分流**：不同群绑到不同项目，同一个机器人即可覆盖你名下的多个项目；群会话按路由到的目标项目分别展示，不会串味。

> 绑定关系集中在 **已配置的绑定** 区管理：可随时查看每条 chat 与 project 的对应关系，并**解除绑定**。

## 第三步：在群里对话式协作

绑定完成后，群就是一个入口：

- **提需求**：在群里 @ 机器人或按平台规则发消息，牛牛会在对应项目起 workspace 让 agent 执行。
- **双向对话**：agent 的真实回复会回到群里；你可以继续追问、纠偏，形成来回对话，而不是"发出去就没下文"。
- **图文入站**：可以直接把截图、日志文件发进群，agent 一并读取。
- **markdown 出站**：agent 的回复以 markdown 渲染，代码块、列表清晰可读。
- **会话切换**：一个群里可能有多个进行中的会话，可按**会话 ID**切换，把回复接到正确的上下文。
- **回执与任务状态**：消息送达有回执；任务进行中的工作提示以各平台原生 emoji 呈现（例如钉钉的忙碌表情）。

## 典型场景

- **随手提单**：在研发群里 @ 机器人"把登录页的报错日志排查一下"，牛牛在对应项目起 workspace，agent 排查后把结论回到群里。
- **值班响应**：告警群绑到运维项目，收到告警后直接在群里让 agent 拉日志、给初步定位。
- **跨项目一机通吃**：一个机器人同时服务"前端""后端""运维"三个群，各自分流到对应项目，互不干扰。
- **移动端驱动**：人在手机上，用飞书 / 钉钉就能给 agent 派活、看进展，回到电脑前再细看。

## 凭据与绑定管理

- **查看绑定**：**设置 → IM 机器人 → 已配置的绑定**，逐条列出 chat ↔ project。
- **解除绑定**：在该列表直接删除某条绑定，之后这个群的消息不再落到原项目。
- **channel_id**：新建 / 配对成功后页面会展示 `channel_id`，作为群的唯一标识，便于排查路由问题。

## 故障排查

| 现象 | 原因 / 处理 |
|---|---|
| 机器人不在线 | 凭据填错或平台侧未启用——重填凭据，点 **测试** 验证长连接 |
| 群里 @ 没反应 | 该群还没绑定项目——去 **已配置的绑定** 确认，或重新配对 |
| 消息落错项目 | 群绑定到了错误项目——解除后重新绑定到正确项目 |
| 图片 / 附件没被读取 | 确认平台机器人已授予读取消息附件的权限 |
